## 递归的暴力美学

作为一门暴力穷举的艺术，回溯的核心本质其实就是**带有“撤销”操作的深度优先搜索（DFS）**。

我们可以把它想象成**走迷宫**：你站在一个路口，面前有几条路（**选择**）。你选了其中一条走下去（**递归**），如果发现这是一条死胡同，或者已经找到了出口（**触发结束条件**），你就必须退回到上一个路口（**撤销选择/回溯**），然后去尝试其他的路。

## 核心要领：灵魂三问

在写任何一道回溯题之前，先在脑海里（或者草稿纸上）画一棵决策树，并回答以下三个问题：

1. **路径（Path）**：我已经做出了哪些选择？（通常是一个列表，记录当前走过的轨迹）。
2. **选择列表（Choices）**：当前我站在这个节点，还有哪些选项可以选？（通常是需要遍历的数组或状态）。
3. **结束条件（Termination）**：什么时候我应该停下来收集结果并往回退？（比如到达了决策树的叶子节点，或者路径长度等于数组长度）。

## 万能解题模板

```py
result = [] # 存放最终结果的全局集合

def backtrack(路径, 选择列表):
    if 满足结束条件:
        # 【极其重要】这里必须存入路径的拷贝/快照！
        result.append(路径[:]) 
        return
        
    for 选择 in 选择列表:
        if 该选择不合法 (例如已经被选过了，或者需要剪枝):
            continue
            
        # 1. 做选择 (改变状态)
        路径.append(选择)
        将该选择从当前可用选项中标记为已使用 (可选)
        
        # 2. 递归进入下一层决策树
        backtrack(路径, 更新后的选择列表)
        
        # 3. 撤销选择 (回溯，恢复状态)
        路径.pop()
        将该选择恢复为可用状态 (可选)
```
