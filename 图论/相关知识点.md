## 图的搜索

### 深度优先搜索 (DFS: Depth-First Search)

**💡 核心思想：“不撞南墙不回头”**

想象你正在走迷宫。DFS 的策略是：选择一条路，一直往前走，走到死胡同（或者所有相邻节点都访问过）为止。然后退回上一个路口，尝试另一条没走过的路，直到所有角落都被探索完毕。

**🛠️ 技术实现与细节**

- **数据结构依赖：** **栈 (Stack)**。通常我们通过递归 (Recursion)来实现，因为函数调用的底层本身就是一个栈。
- **时间复杂度：** $O(V + E)$ （$V$ 是顶点数，$E$ 是边数。每个节点和每条边都会被访问一次）。
- **空间复杂度：** $O(V)$ （最坏情况下，图退化成一条链，递归调用栈的深度为 $V$）。
- **典型应用场景：**
  - 迷宫寻路、解数独（回溯法）。
  - 检测图中是否存在环。
  - 拓扑排序。

代码实现：

```py
def dfs(graph, start, visited=None):
    """
    深度优先搜索 (递归实现)
    :param graph: 图的邻接表表示 (字典)
    :param start: 起始节点
    :param visited: 记录已访问节点的集合，防止死循环
    """
    if visited is None:
        visited = set()
    
    # 标记当前节点为已访问，并处理(这里以打印为例)
    visited.add(start)
    print(start, end=" -> ")

    # 遍历当前节点的所有邻居
    for neighbor in graph[start]:
        # 如果邻居没有被访问过，则"撞南墙不回头"地继续深入
        if neighbor not in visited:
            dfs(graph, neighbor, visited)
```



### 广度优先搜索 (BFS: Breadth-First Search)

**💡 核心思想：“一石激起千层浪”**

想象你在平静的湖面投入一颗石子。BFS 的策略就像水波一样，以起点为中心，一圈一圈地向外扩展。先拜访离起点最近的所有邻居，然后再拜访邻居的邻居，层层递进。

**🛠️ 技术实现与细节**

- **数据结构依赖：** **队列 (Queue)**。遵循“先进先出”的原则，保证先被发现的节点先被探索。
- **时间复杂度：** $O(V + E)$ （同 DFS，遍历所有的顶点和边）。
- **空间复杂度：** $O(V)$ （最坏情况下，队列中需要存储同一层的所有节点，对于极其“宽”的图，空间开销较大）。
- **典型应用场景：**
  - 无权图的最短路径问题（例如社交网络中“你和某人隔着几度连接”）。
  - 网络爬虫（逐层抓取网页）。
  - 寻找最近的兴趣点。

代码实现：

```py
from collections import deque

def bfs(graph, start):
    """
    广度优先搜索 (迭代实现)
    :param graph: 图的邻接表表示 (字典)
    :param start: 起始节点
    """
    visited = set()          # 记录已访问节点
    queue = deque([start])   # 初始化队列，并将起点加入
    
    visited.add(start)       # 标记起点为已访问

    while queue:
        # 1. 节点出队
        vertex = queue.popleft()
        print(vertex, end=" -> ")

        # 2. 遍历该节点的所有邻居
        for neighbor in graph[vertex]:
            if neighbor not in visited:
                # 3. 将未访问的邻居标记为已访问，并加入队列尾部，等待后续探索
                visited.add(neighbor)
                queue.append(neighbor)
```



### DFS 与 BFS 对比总结

| **特性**     | **深度优先搜索 (DFS)**     | **广度优先搜索 (BFS)**       |
| ------------ | -------------------------- | ---------------------------- |
| **探索方式** | 纵深探索，深入到底         | 分层探索，齐头并进           |
| **数据结构** | 栈 (Stack) / 递归          | 队列 (Queue)                 |
| **最短路径** | **不能**保证找到最短路径   | 在无权图中能找到**最短路径** |
| **内存消耗** | 通常较小（取决于图的深度） | 可能较大（取决于图的宽度）   |

## 判断图是否有环

### 有向图

在有向图中，边是有方向的。假设我们有一个包含四个节点的图，正常的信息流向可能是 $V_1 \to V_2 \to V_3 \to V_4$。但如果在梳理时发现 $V_4$ 到 $V_1$ 存在一条反向边，即 $V_4 \to V_1$，这就首尾相连构成了一个非常典型的有向环。

对于这类图，最常用的检测算法有两种：

1. DFS + 三色标记法
2. 拓扑排序（基于BFS）

#### DFS + 三色标记法

在深度优先搜索的过程中，为每个节点标记三种不同的状态：

- 白色：尚未被访问过
- 灰色：正在访问中（当前节点及其后代节点还在递归栈中，尚未回溯）
- 已经完全访问完毕（当前节点的所有相连出度路径都已探索完毕）

**核心判断逻辑**：在 DFS 的遍历过程中，如果我们试图访问一个相邻的节点，并且发现它的状态是**灰色**，这就意味着我们顺着边走回到了当前递归栈中的某个祖先节点，**说明图中存在环**。

```py
def has_cycle_directed_dfs(num_nodes: int, edges: list[tuple[int, int]]) -> bool:
    # 1. 构建邻接表
    adj = {i: [] for i in range(num_nodes)}
    for u, v in edges:
        adj[u].append(v)
        
    # 2. 初始化颜色数组
    # 0: 未访问 (White), 1: 正在访问 (Gray), 2: 已完全访问 (Black)
    colors = [0] * num_nodes
    
    def dfs(node):
        colors[node] = 1  # 标记为正在访问
        
        for neighbor in adj[node]:
            if colors[neighbor] == 1:
                return True  # 遇到灰节点，发现环
            if colors[neighbor] == 0:
                if dfs(neighbor):
                    return True
                    
        colors[node] = 2  # 当前节点及其所有出度均访问完毕，标黑
        return False

    # 3. 遍历所有节点（处理非连通图的情况）
    for i in range(num_nodes):
        if colors[i] == 0:
            if dfs(i):
                return True
                
    return False
```

#### 拓扑排序（基于BFS）

拓扑排序的本质是不断剥离图中“没有前置依赖”的节点。

- **执行步骤**：
  1. 统计所有节点的入度（In-degree，即指向该节点的边数）。
  2. 将所有入度为 $0$ 的节点加入一个队列。
  3. 当队列不为空时，弹出一个节点，并把该节点指向的所有邻接节点的入度减 $1$。如果某个邻接节点的入度随之变为 $0$，则将其加入队列。
  4. 记录弹出的节点总数。
- **核心判断逻辑**：当队列为空时，如果记录的弹出节点总数 **小于** 图中的总节点数，**说明图中存在环**。因为环中的节点互相依赖，彼此的入度永远无法减小到 $0$，所以永远进不了队列。

```py
from collections import deque

def has_cycle_directed_kahn(num_nodes: int, edges: list[tuple[int, int]]) -> bool:
    adj = {i: [] for i in range(num_nodes)}
    in_degree = [0] * num_nodes
    
    # 1. 构建邻接表并统计各节点入度
    for u, v in edges:
        adj[u].append(v)
        in_degree[v] += 1
        
    # 2. 将所有入度为 0 的节点加入队列
    queue = deque([i for i in range(num_nodes) if in_degree[i] == 0])
    visited_count = 0
    
    # 3. BFS 剥离节点
    while queue:
        node = queue.popleft()
        visited_count += 1
        
        for neighbor in adj[node]:
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)
                
    # 4. 如果出队的节点数不等于图的总节点数，说明存在环
    return visited_count != num_nodes
```

### 无向图

在无向图中，边是没有方向的（节点 $A$ 和 $B$ 连通，既意味着 $A$ 可以到 $B$，$B$ 也可以到 $A$）。

**并查集**是处理无向图连通性最优雅的数据结构，非常适合用来做动态的环路检测。

- **执行步骤**：
  1. 初始化时，每个节点各自为一个独立的集合（自己的父节点是自己）。
  2. 遍历图中的每一条边 $(u, v)$。
- **核心判断逻辑**：在处理边 $(u, v)$ 时，去查找 $u$ 和 $v$ 的根节点（即判断它们是否属于同一个集合）。如果它们在合并前就已经拥有相同的根节点，**说明图中存在环**；如果不同，则将 $u$ 和 $v$ 所在的集合合并。

```py
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        
    def find(self, i):
        # 路径压缩，直接指向根节点
        if self.parent[i] != i:
            self.parent[i] = self.find(self.parent[i])
        return self.parent[i]
        
    def union(self, i, j):
        root_i = self.find(i)
        root_j = self.find(j)
        
        if root_i == root_j:
            return False  # 发现环：两个节点已经在同一个集合中
            
        self.parent[root_i] = root_j
        return True

def has_cycle_undirected_uf(num_nodes: int, edges: list[tuple[int, int]]) -> bool:
    uf = UnionFind(num_nodes)
    for u, v in edges:
        if not uf.union(u, v):
            return True
    return False
```
