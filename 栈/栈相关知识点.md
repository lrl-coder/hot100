# 算法通关秘籍：栈 (Stack) 核心知识与刷题指南

欢迎来到栈的算法特训！栈是算法面试中最常考、最能考察逻辑思维的数据结构之一。这份文档为你梳理了栈的核心知识点、常见题型套路以及必刷经典题，非常适合作为你刷题时的速查手册。

---

## 1. 栈的核心概念 (The LIFO Principle)

栈是一种**后进先出 (LIFO, Last In First Out)** 的线性数据结构。你可以把它想象成食堂里叠在一起的一摞盘子：你只能从最上面拿走盘子（出栈），也只能把新盘子放在最上面（入栈）。

这种“后来居上”的特性，使得栈非常适合处理**具有依赖关系、需要反向追溯**或者**需要匹配**的问题。

---

## 2. 基本操作与时间复杂度

对于一个标准实现（如基于数组或链表）的栈，其核心操作的复杂度都极低，这也是它在算法中大放异彩的原因。

| 操作接口           | 功能描述             | 时间复杂度 | 空间复杂度 |
| ------------------ | -------------------- | ---------- | ---------- |
| **Push(x)**        | 将元素 x 压入栈顶    | O(1)       | O(1)       |
| **Pop()**          | 弹出并返回栈顶元素   | O(1)       | O(1)       |
| **Peek() / Top()** | 查看栈顶元素但不弹出 | O(1)       | O(1)       |
| **isEmpty()**      | 判断栈是否为空       | O(1)       | O(1)       |

> **教学提示：** 在很多高级语言中（如 Python 的 `list`，C++ 的 `std::stack`，Java 的 `ArrayDeque`），栈的底层通常由动态数组实现。虽然在触发扩容时 Push 操作的时间复杂度偶尔会退化，但其**均摊时间复杂度依然是 O(1)**。

## 3. 核心题型与解题套路

在刷题时，栈的应用通常可以归纳为以下四大类。掌握了这些套路，就能看透 80% 的栈相关题目。

### 题型一：成对匹配问题

* **特征：** 题目涉及括号、消消乐、相邻字符消除等。
* **解题套路：** 遍历元素时，将当前元素与栈顶元素进行比对。如果匹配（如左括号遇到右括号，或者两个相同的字符相撞），则将栈顶元素 `Pop`；如果不匹配，则将当前元素 `Push`。
* **经典例题：** LeetCode 20 (有效的括号), LeetCode 1047 (删除字符串中的所有相邻重复项)。

### 题型二：单调栈 (Monotonic Stack) —— **核心重难点**

* **特征：** 题目通常要求在数组中寻找 **“下一个更大元素”** 或 **“下一个更小元素”**，或者涉及区间最大/最小值计算（如接雨水、最大矩形面积）。
* **解题套路：** 维护一个栈，使得栈内元素（通常存的是数组的**索引**）保持严格单调递增或单调递减。
* **找下一个更大元素：** 维护一个**单调递减栈**。当新元素大于栈顶元素时，说明找到了栈顶元素的“下一个更大值”，不断弹出栈顶并记录答案，直到新元素小于等于栈顶，然后将新元素入栈。
* **找下一个更小元素：** 维护一个**单调递增栈**。


* **经典例题：** LeetCode 739 (每日温度), LeetCode 496 (下一个更大元素 I)。

### 题型三：表达式解析与计算

* **特征：** 处理数学表达式、逆波兰表达式（后缀表达式）、目录路径简化等。
* **解题套路：** 遇到数字或路径名则入栈；遇到操作符（如 `+`, `-`, `*`, `/`）或特定指令（如 `..` 返回上一级）则从栈中弹出指定数量的元素，计算/处理后再将结果压回栈中。
* **经典例题：** LeetCode 150 (逆波兰表达式求值), LeetCode 71 (简化路径)。

### 题型四：深度优先搜索 (DFS) 与递归模拟

* **特征：** 树或图的深度优先遍历，或者题目要求显式地将递归代码转化为迭代代码。
* **解题套路：** 计算机系统底层本身就是用“调用栈”来实现递归的。你可以手动创建一个栈，按顺序压入节点或状态，从而完美模拟递归过程。
* **经典例题：** 树的前/中/后序遍历（迭代版）。

## 5. 刷题心法总结

1. **善存索引：** 在处理单调栈问题时，栈里**尽量存元素的索引，而不是元素的值**。有了索引，你随时可以通过数组拿到值，并且还能顺便算出两个元素之间的距离（宽度）。
2. **前置哨兵（Dummy Node/Sentinel）：** 在做柱状图最大矩形等题目时，可以在数组头尾加上高度为 0 的“哨兵”，这样可以避免复杂的边界情况判断，让所有元素都能顺利出栈。
3. **画图推演：** 遇到复杂的栈变动（尤其是求表达式或接雨水），不要纯靠脑补。在草稿纸上画一个竖直的 U 型桶，手动模拟 3-4 个元素的入栈出栈过程，逻辑就会瞬间清晰。
